### HIGH

1. **KrakenCLI._run swallows non-zero exit codes with non-empty stdout** — `hydra_agent.py:144-152`. If `returncode != 0` but stdout has bytes, we attempt JSON parse and may mis-classify partial output as success. Only the empty-stdout branch surfaces the exit code. *Fix:* check `result.returncode` before parsing and return an explicit error dict.
2. **DerivativesStream subprocess errors logged only into `fetch_errors` counter** — `hydra_derivatives_stream.py:226-227`. A sustained WSL/kraken-CLI failure silently degrades indicators; only R10 (staleness) catches it downstream. *Fix:* emit a single stderr warning when `fetch_errors[pair] >= 3` and expose the streak on `DerivativesSnapshot` so Quant sees "repeatedly failing," not just "stale."
3. **R8 contrarian-edge wiring depends on LLM returning valid `positioning_bias`** — `hydra_brain.py:641` → `hydra_agent.py:3194-3195` → `hydra_quant_rules.py:147`. The plumbing is correct (decision.positioning_bias is populated from analyst_output), but if the Quant LLM returns an empty/missing `positioning_bias` field, R8 never fires. *Note:* this is the intended contract — no signal → no boost — but we should add a fallback heuristic or log a counter tracking how often the Quant returns a valid value, so we can tell whether R8 is earning its keep. **Downgrade candidate to MEDIUM** depending on live-data observation.

### MEDIUM

1. `hydra_engine.py:1330` — `cvd_divergence_sigma` returns once `len(diffs) >= 4`; with 3 historical samples the `pstdev` is weak. Raise the gate to `>= 8`.
2. `hydra_engine.py:1905, 1932` — `restore_runtime` silently drops malformed trade/candle rows. Log a single warning per dropped row so corrupted snapshots are visible.
3. `hydra_brain.py:1393-1401` — `_log_jsonl` has no `parent.mkdir`; a nonexistent log directory silently loses the audit trail. Create the parent dir at `__init__` (where `_jsonl_path` is set) and keep the IO-error swallow.
4. `hydra_brain.py:985-998` vs `355-363` — `_coerce_size_mult` and the RM clamp path both cap to `[0, 1.5]` independently with asymmetric observability. Consolidate or print on clamp.
5. `hydra_brain.py:302-323` — `TOOLS_GUIDANCE` lists tool names that must match `BACKTEST_TOOLS` keys; no validation. Add an `assert` at `__init__` so drift fails loud.
6. `hydra_agent.py:3150-3159` — W3 `api_down_block` blocks BUY but passes SELL through unmodified. Original signal reason is overwritten; preserve `original_reason` for audit.
7. `hydra_agent.py:3183-3212` — `apply_rules()` is skipped on `fallback=True` or `api_down_block=True`. R10 (staleness) should still fire in those states. Run rules always; let R10 see the flag.
8. `hydra_agent.py:2698-2699, 3220-3248` — `_last_ai_decision` cache replayed on HOLD pairs with no "cached_from_tick" marker → dashboard renders stale decision as current. Tag the payload.
9. `hydra_agent.py:1858-1876` — teardown order stops execution/candle streams *before* `derivatives_stream`; fills mid-shutdown may read stale quant_indicators. Stop `derivatives_stream` first.
10. `hydra_agent.py:3214-3218` — size-multiplier clamp ∈ [0, 1.5] is silent. Emit `size_multiplier_clamped_from` when the pre-clamp product exceeds 1.5.
11. `dashboard/src/App.jsx:4380+` — **v2.14 disclosure fields are exported by the agent but not rendered.** `size_multiplier_brain`, `size_multiplier_rules`, `rules_triggered`, `rules_force_hold`, `rules_force_hold_reason` all present in `state["ai_decision"]` but no UI surface. Add a collapsible "Size breakdown" block under the QUANT reasoning.
12. `tests/test_engine_cvd.py:134, 165` & `tests/test_derivatives_stream.py:177` — time-dependent or tautological assertions. Freeze clocks; strengthen `isinstance(sigma, float) or None` into a real assertion under seeded input.
