## Verified HIGH issues + fixes

### 1. `BalanceStream._on_message` unguarded `float(balance)` — `hydra_agent.py:1083`
A malformed balance value from the WS feed would raise `ValueError`,
crash the reader thread, and force a stream restart. **Fix:** wrap in
try/except, skip the entry on coercion failure.

### 2. `BookStream._on_message` unguarded `float()` in nested level loop — `hydra_agent.py:1019,1023`
Same risk pattern; one bad bid/ask level would kill the whole stream.
**Fix:** factored a `_as_level` helper that returns `None` on coercion
failure; bad levels are skipped, good ones stored.

### 3. CI gate missing 23 test files
`.github/workflows/ci.yml` ran only 22 of 45 `tests/test_*.py` files. The
omissions included **`test_backtest_drift.py`** (the I7 invariant gate
explicitly named in CLAUDE.md §Backtesting safety invariants) and
**`test_partial_fill_reconcile.py`** (HF-005 fix coverage). **Fix:** added
all 24 missing test files (plus the 2 new error-path test files written
this session — see below) as named CI steps with `-v` verbosity. CI now
runs all 47 test files.

### 4. Live-execution mock harness 33/35 failing on `main` (pre-existing)
Surfaced when running Phase 1 self-audit. `hydra_companions/config.py`
calls `_load_env_once()` at module-import time, reading `.env` and
re-populating `XAI_API_KEY` / `ANTHROPIC_API_KEY` after the harness's
`isolate_environment()` had popped them. Scenario #1 ran before the lazy
companion-config import, saw a clean env. Scenarios #2+ inherited the
re-populated env, the brain was constructed, and the per-scenario
`assert agent.brain is None` failed. **Fix:**
- Added `HYDRA_NO_DOTENV=1` sentinel honored by `_load_env_once`
- Harness sets the sentinel during `isolate_environment` and restores
  the prior value during `restore_environment`
- Result: **35/35 mock harness scenarios pass** (was 2/35).

## Verified MED issues + fixes

### 5. `compiler.py` `souls_dir: Optional[Path]` rejected str inputs — `hydra_companions/compiler.py:406,417`
Type annotation said `Path` but `(souls_dir or SOULS_DIR) / ...` raised
`TypeError` if a caller passed a plain string. **Fix:** wrap with
`Path(souls_dir) if souls_dir is not None else SOULS_DIR` in both
`load_soul` and `load_all_souls`.

### 6. Test coverage gap on new error paths
The earlier session added typed-error guarantees to:
- `hydra_companions/compiler.py` (`load_all_souls` skips bad souls;
  `load_soul` raises `RuntimeError`)
- `hydra_journal_migrator.py` (typed `RuntimeError` with file path)

None of these had test coverage. **Fix:** wrote two new test files —
`tests/test_companion_compiler_errors.py` (6 tests covering corrupt JSON,
missing fields, str-path acceptance, RuntimeError propagation) and
`tests/test_journal_migrator_errors.py` (4 tests covering corrupt legacy
journal, corrupt session snapshot, clean-dir no-op, idempotent skip).
Both files added to CI gate. **10/10 pass.**
