## HIGH (fixed)

1. **`hydra_backtest.py:614` — `_validate_brain_mode()` defined but never called.**
   `BacktestRunner.__init__` did not invoke the brain-mode guard. A caller passing
   `brain_mode="live"` or `"replay"` would silently fall through to stub-equivalent
   behavior with no error, defeating the v2.20.0 MVP scoping contract.
   **Fix:** Added `self._validate_brain_mode()` call in `__init__` before
   `_build_engines_and_coord()`. Two existing tests updated to expect throw at
   construction rather than direct method call.

2. **`.github/workflows/ci.yml` — 13 new tests not wired into CI.**
   Tests existed in `tests/` but the engine-tests workflow job did not invoke them,
   so they ran locally only. Coverage gap on every v2.20.0 regression path:
   history store, walk-forward, Wilcoxon, tape capture, sqlite source, research
   WS handlers, regression runner, bootstrap, refresh.
   **Fix:** Added a `Run v2.20 research tab tests` step before the live-execution
   harness, invoking pytest on all 13 files in one batch.

## MEDIUM (fixed)

3. **`tools/bootstrap_history.py:134` — `--out` did not respect `HYDRA_HISTORY_DB` env.**
   Inconsistent with `tools/run_regression.py` which honored the env var.
   **Fix:** Default now `os.environ.get("HYDRA_HISTORY_DB", "hydra_history.sqlite")`.

4. **`tools/refresh_history.py:122` — `--db` did not respect `HYDRA_HISTORY_DB` env.**
   Same inconsistency.
   **Fix:** Added `import os`; default now reads env var.
